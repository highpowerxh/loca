Locality Lab Installation

 

  1. Install Pin, GCC, Ruby (or JRuby), RScript etc. according to instructions on the following page.

  2. Unpack the directory.  Let [dir] be the root directory.  The client is in [dir]/gem/lib.  The server is in [dir]/server.

  3. To setup the environment, you need to run ‘[dir]/gem/lib/loca config’. It asks two questions: the first is the path of the Pin tool, e.g. /localdisk/cding/pin/pin-2.13-62732-gcc.4.4.7-linux/. The second is the host and path of the data repository. Now we use the same machine, simply use the default (by skipping the answer).

  4. To test a program, run ‘[dir]/gem/lib/loca run [-d/-i/-a]’ followed by the program command and parameters. -d is default and means data analysis, -i means instruction analysis, and -a will do both, e.g. ‘[dir]/gem/lib/loca run gzip big_file’ or ‘[dir]/gem/lib/loca run -d gzip big_file’ will run data analysis on gzip big_file, while ‘[dir]/gem/lib/loca run -i gzip big_file’ will run instruction analysis.

  5. To test a parallel program, run ‘[dir]/gem/lib/loca prun’ followed by the program command and parameters. After a 'prun', use '[dir]/gem/lib/loca pfp' to get the footprint of a previous prun or followed by the name of a specified prun. Then you can use '[dir]/gem/lib/loca pmr' to get the miss ratio.

  6. To browse the result, first run ‘[dir]/server/server’. It will serves from current dir if it named 'data' or the same data path (~/.loca/data) with the client if you run it locally.

  7. After reporting result to local server with ‘[dir]/gem/lib/loca upload’(not needed if the server serves from ~/.loca/data), or remote server with ‘[dir]/gem/lib/loca upload -r’, the result will be plotted in a pop-up window of your default browser.
